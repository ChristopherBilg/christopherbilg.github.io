What this demonstrates

  • Facade / Registry — Ontology is the single entry point; object & link types are registered declaratively.
  • Adapter — each object type binds to a backing JSON source via a backingData URL and primary key.
  • Proxy — ObjectProxy lazily merges base rows with kinetic edits on every property read.
  • Command — every mutation goes through ActionEngine.dispatch; validation, state machine, and side effects are encapsulated in the action spec.
  • Memento — ChangeSets persist to localStorage; each is addressable and reversible.
  • Observer — UI subscribes to action/undo/loaded events; no direct DOM coupling to mutations.
  • State — Flight.status transitions are gated by the action validators (Scheduled → InAir → Landed, with Cancelled as an early exit).
  • Apollo-sim — manifest.json feature flags gate which actions are exposed, mimicking environment-versioned rollout.
